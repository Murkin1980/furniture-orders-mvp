# Source Notes

## UPROCK: SaaS interface design

Source:
https://www.uprock.ru/articles/dizayn-interfeysov-dlya-saas-produktov

Useful principles:

- Function and reduced cognitive load come before visual novelty.
- Navigation should make broad functionality easy to find.
- Modular design supports product growth.
- Mobile must support real tasks, not only show a compressed layout.
- Data presentation should help users make decisions.
- Integrations and feedback loops matter when they serve product needs.

Important qualification:

- Personalization, themes, charts, pricing, integrations, and feedback widgets
  are not universal requirements. Add them only after a concrete workflow,
  business model, or user need justifies them.
- Trust is created primarily through predictable behavior, clear permissions,
  status visibility, error prevention, and recovery. Security-colored icons
  alone do not create trust.
- Lists of AI design tools age quickly and should not define the workflow.

## Nielsen Norman Group: usability heuristics

Source:
https://www.nngroup.com/articles/ten-usability-heuristics/

Apply these cross-cutting checks:

- visibility of system status;
- match with the real world;
- user control and freedom;
- consistency and standards;
- error prevention;
- recognition rather than recall;
- flexibility and efficiency;
- focused minimalist design;
- clear error diagnosis and recovery;
- contextual help.

## W3C WCAG 2.2 Quick Reference

Source:
https://www.w3.org/WAI/WCAG22/quickref/

Use it for implementation verification, especially:

- semantic relationships and meaningful sequence;
- contrast and non-color status cues;
- reflow and text spacing;
- keyboard access and visible, unobscured focus;
- headings and labels;
- target size;
- predictable navigation and identification;
- error identification, suggestions, and prevention.
