## Главные замечания

### 1. Подтвердить синтаксис исходных файлов

По отображению вложений `orders.js`, `orders-core.js` и `orders-core.test.js` выглядят так, будто в них местами пропали закрывающие скобки и части объектов. Это может быть проблемой отображения вложений, а может быть и реальной проблемой в файлах. [file:134][file:135][file:138]

Что сделать:

- локально выполнить:

```bash
npm run check
npm test
```

- если есть ошибки синтаксиса, сначала исправить их, а уже потом двигаться дальше. [file:133]

### 2. Зафиксировать имя D1 binding как каноническое

Сейчас в проекте binding называется `DB`, и это имя уже используется в `wrangler.toml` и README. Нужно оставить только это имя во всех файлах проекта и не смешивать его с альтернативами вроде `ORDERS_DB`. [file:128][file:132]

### 3. Добавить явную структуру проекта в README

README уже хороший, но ему не хватает краткого дерева проекта. Это упростит онбординг и последующую передачу проекта в работу или расширение под другие мебельные компании. [file:128]

Рекомендуемый блок:

```text
furniture-orders-mvp/
  functions/api/orders.js
  src/orders-core.js
  public/index.html
  migrations/0001_orders.sql
  tests/orders-core.test.js
  wrangler.toml
  package.json
```

### 4. Не держать `ensureSchema()` как долгосрочную прод-стратегию

Для локального режима и MVP вызов `ensureSchema(db)` допустим, но в продакшне схема должна закрепляться миграциями D1. Runtime-создание таблиц лучше оставить как временную страховку для dev, а не как основной способ инициализации базы. [file:128][file:135][file:136]

### 5. Уточнить и локализовать Telegram-сообщение

Сейчас Telegram-уведомление по логике уже есть, что хорошо. Но для универсального шаблона под мебельные компании лучше сразу привести текст уведомления к более прикладному русскоязычному или локализуемому формату. [file:135]

Пример полезнее текущего нейтрального сообщения:

```text
Новая заявка на мебель
Заказ: #123
Клиент: Ерлан
Телефон: +77011234567
Город: Алматы
Тип: kitchen
Бюджет: 850000
Комментарий: Нужна угловая кухня
Источник: site
```

### 6. Убедиться, что форма реально совпадает с backend-контрактом

По `index.html` видно правильное направление: форма отправляет в `/api/orders` и ориентирована на единый контракт заявки. Но нужно вручную проверить, что `name` атрибуты полей действительно совпадают с тем, что ожидает `normalizeOrderPayload()` в `orders-core.js`: `name`, `phone`, `source`, `city`, `furnitureType`, `budget`, `description`. - форма отправляет заявку через UI; [file:137]

- Telegram-уведомление приходит при заданных env-переменных; [file:128][file:135]
- секреты не захардкожены в коде. [file:128][file:135]
