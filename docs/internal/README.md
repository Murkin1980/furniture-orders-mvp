# Internal Project History

This directory keeps historical implementation instructions, coding briefs,
handoffs, and reviewer material out of the repository root.

- `stages/` contains the archived Stage 1-4 implementation history.
- `notebooklm/` contains compressed context prepared for external research
  tools.
- Current session summaries remain in `docs/sessions/`.

These documents are useful for audits and continuation, but they are not the
primary entry point for development. Start with the root `README.md`,
`PRODUCT.md`, `AGENTS.md`, and `SESSION_NOTES.md`.
