# AI infrastructure stage 1 coding brief

## Goal

Implement the low-risk first stage from `CODEX_AI_INFRA_INSTRUCTIONS.md` in the main product repository.

## Scope

- Add missing project-control documents.
- Add a compact Markdown knowledge base and reusable project skills.
- Record verified external-tool decisions.
- Install and build a local CodeGraph index.
- Install MarkItDown locally for future document conversion.

## Boundaries

- Do not add Supermemory or Headroom to application runtime yet.
- Do not send client data, source code, or documents to external services.
- Do not invent prices, legal terms, company facts, or WhatsApp scripts.
- Do not modify production settings, migrations, endpoints, or UI behavior.
- Preserve existing uncommitted documentation changes.

## Verification

- Build local CodeGraph index.
- Verify MarkItDown CLI.
- Run `npm.cmd test`, `npm.cmd run check`, and `git diff --check`.
